<span style="color:#002060">Coroutines are a powerful tool in unity as an</span>  <span style="color:#002060"> __alternative to threads\.__ </span>

<span style="color:#002060">The main purpose is to be able</span>  <span style="color:#002060"> __to run any method in the same way as with the Update\.__ </span>

<span style="color:#002060">I mean\, first we can register a method\. Then we will call that method in each frame\.</span>

<span style="color:#002060">We can start and stop them using special instructions like</span>  <span style="color:#002060"> __StartCoroutine__ </span>  <span style="color:#002060">and</span>

<span style="color:#002060"> __StopCoroutine__ </span>  <span style="color:#002060">\.</span>

<span style="color:#002060">Unity 3D will save the list of all the active coroutines which will invoke after the</span>  <span style="color:#002060"> __LateUpdate__ </span>  <span style="color:#002060">of each object in the scene\.</span>

<span style="color:#002060">So you're going to call these coroutines: They will be run</span>  <span style="color:#002060"> __after the LateUpdate every frame\.__ </span>

<span style="color:#002060">The coroutines must carry out a</span>  <span style="color:#002060"> __small action__ </span>  <span style="color:#002060">and return control immediately\.</span>

<span style="color:#002060">When the coroutine is called again\,</span>  <span style="color:#002060"> __the following instruction__ </span>  <span style="color:#002060">will be executed after the transfer of control by unity\.</span>

<span style="color:#002060">Its implementation in C \# will be carried out using</span>  <span style="color:#002060"> __yield return__ </span>  <span style="color:#002060">\.</span>

<span style="color:#002060">The coroutines will return</span>  <span style="color:#002060"> __Ienumerator__ </span>  <span style="color:#002060">and each</span>  <span style="color:#002060"> __yield return__ </span>  <span style="color:#002060">will get the control to Unity until it is invoked again in the next frame continuing its execution after the last yield return\.</span>
